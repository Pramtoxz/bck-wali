CARA MENDAPATKAN KOORDINAT DARI GOOGLE MAPS
=====================================

Metode 1: Via Web Interface (Paling Mudah)
-------------------------------------------
1. Login sebagai admin
2. Buka menu "Lokasi Kantor"
3. Klik "Tambah Lokasi" atau "Edit" pada lokasi yang ada
4. Klik tombol "Buka Google Maps" atau "Lihat di Google Maps"
5. Di Google Maps:
   - Cari lokasi kantor yang diinginkan
   - Klik kanan pada titik lokasi di peta
   - Klik koordinat yang muncul (contoh: -0.9492, 100.3543)
   - Koordinat akan otomatis tersalin ke clipboard
6. Paste koordinat di form (latitude dan longitude)
7. Set radius sesuai kebutuhan (dalam meter)
8. Centang "Aktifkan lokasi ini" jika ingin mengaktifkan
9. Klik "Simpan"

Metode 2: Manual dari Google Maps
----------------------------------
1. Buka https://www.google.com/maps
2. Cari lokasi kantor (contoh: "Kantor Wali Nagari Padang Panjang")
3. Klik kanan pada titik lokasi yang tepat
4. Klik koordinat yang muncul di popup (format: -0.9492, 100.3543)
5. Koordinat akan tersalin otomatis
6. Paste di form atau API request

Format Koordinat:
-----------------
- Latitude: -90 sampai 90 (contoh: -0.9492)
- Longitude: -180 sampai 180 (contoh: 100.3543)
- Gunakan titik (.) bukan koma (,) untuk desimal
- Presisi 6 digit desimal sudah cukup akurat

Tips:
-----
- Zoom in ke lokasi untuk mendapatkan koordinat yang lebih akurat
- Pastikan titik yang dipilih adalah pintu masuk kantor
- Radius 20-50 meter biasanya cukup untuk area kantor
- Hanya satu lokasi yang bisa aktif untuk validasi absensi
- Koordinat di tabel bisa diklik untuk membuka Google Maps

=====================================
TROUBLESHOOTING - ERROR "UNAUTHENTICATED"
=====================================

Error Message:
--------------
{"success":false,"message":"Unauthenticated. Please login via POST /api/auth/login to get a token, then set Authorization: Bearer {token} header.","hint":"API routes require Bearer token authentication. Web routes use session cookies."}

Penyebab dan Solusi:
--------------------

1. MENGAKSES API ENDPOINT DARI BROWSER
   Penyebab: Membuka URL seperti http://localhost:8000/api/office-locations di browser
   Solusi: 
   - Untuk web interface, gunakan: http://localhost:8000/office-locations (tanpa /api)
   - Untuk testing API, gunakan Postman dengan Bearer token

2. LUPA SET AUTHORIZATION HEADER DI POSTMAN
   Penyebab: Request API tanpa token
   Solusi:
   a. Login dulu untuk dapat token:
      POST http://localhost:8000/api/auth/login
      Body: {"username":"admin","password":"admin123"}
   b. Copy token dari response
   c. Di Postman, tab Authorization:
      - Type: Bearer Token
      - Token: paste token yang di-copy
   d. Atau set manual di Headers:
      - Key: Authorization
      - Value: Bearer {token}

3. TOKEN EXPIRED ATAU INVALID
   Penyebab: Token sudah tidak valid
   Solusi: Login ulang untuk mendapatkan token baru

4. SALAH ENDPOINT
   Penyebab: Menggunakan endpoint web untuk API atau sebaliknya
   Solusi:
   - Web (Browser): /office-locations (pakai session cookie)
   - API (Postman/Mobile): /api/office-locations (pakai Bearer token)

Perbedaan Web vs API:
---------------------

WEB ROUTES (Browser):
- URL: http://localhost:8000/office-locations
- Auth: Session cookie (otomatis setelah login via form)
- Login: http://localhost:8000/login (form HTML)
- Response: HTML page (Inertia React)
- Middleware: auth

API ROUTES (Postman/Mobile):
- URL: http://localhost:8000/api/office-locations
- Auth: Bearer token (manual set di header)
- Login: POST http://localhost:8000/api/auth/login (JSON)
- Response: JSON
- Middleware: auth:sanctum

Cara Testing yang Benar:
-------------------------

TESTING WEB INTERFACE:
1. Buka browser
2. Akses: http://localhost:8000/login
3. Login dengan username & password
4. Akses: http://localhost:8000/office-locations
5. Session cookie otomatis terkirim

TESTING API ENDPOINT:
1. Buka Postman
2. Login:
   POST http://localhost:8000/api/auth/login
   Headers: 
     Accept: application/json
     Content-Type: application/json
   Body (raw JSON):
     {"username":"admin","password":"admin123"}
3. Copy token dari response.data.token
4. Set Authorization:
   Type: Bearer Token
   Token: {paste token}
5. Test endpoint:
   GET http://localhost:8000/api/office-locations
   Headers:
     Accept: application/json
     Authorization: Bearer {token}

Quick Fix:
----------
Jika dapat error "Unauthenticated":
1. Pastikan Anda menggunakan tool yang tepat (Browser untuk web, Postman untuk API)
2. Pastikan URL benar (/office-locations untuk web, /api/office-locations untuk API)
3. Untuk API, pastikan sudah login dan set Bearer token
4. Untuk Web, pastikan sudah login via form

Catatan Penting:
----------------
- JANGAN akses URL /api/* di browser (akan error Unauthenticated)
- JANGAN akses URL web di Postman tanpa setup khusus
- Token Sanctum tidak expire secara default (revoke manual via logout)
- Satu user bisa punya multiple token (login dari multiple device)

20. GET ATTENDANCE STATISTICS
=====================================
GET {{base_url}}/attendance/statistics?month=2026-04

Headers:
Accept: application/json
Authorization: Bearer {token}

Query Parameters:
- month: optional, format YYYY-MM (default: current month)

Expected Response (200):
{
    "success": true,
    "data": {
        "period": {
            "month": "April",
            "year": 2026,
            "month_number": 4
        },
        "statistics": {
            "total_working_days": 22,
            "present_days": 18,
            "late_days": 2,
            "absent_days": 1,
            "field_duty_days": 1,
            "leave_days": 0,
            "attendance_rate": 81.82,
            "punctuality_rate": 88.89
        }
    },
    "message": "Statistik absensi berhasil dimuat"
}

Calculation:
- total_working_days: Jumlah hari kerja (Senin-Jumat) dalam bulan
- present_days: Jumlah hari hadir (ada data absensi)
- late_days: Jumlah hari terlambat (check_in > 08:00:00)
- absent_days: working_days - present_days - field_duty_days - leave_days
- field_duty_days: Total hari dinas luar (approved)
- leave_days: Total hari izin/cuti (approved)
- attendance_rate: (present_days / working_days) * 100
- punctuality_rate: ((present_days - late_days) / present_days) * 100

Use Case:
- Dashboard statistik bulanan
- Laporan kehadiran karyawan
- Monitoring kedisiplinan
- Evaluasi performa

=====================================
ATTENDANCE RECAP BUSINESS RULES
=====================================

Working Days:
- Senin sampai Jumat dihitung sebagai hari kerja
- Sabtu dan Minggu tidak dihitung
- Tidak ada perhitungan hari libur nasional (future enhancement)

Late Detection:
- Check in setelah jam 08:00:00 dianggap terlambat
- Status berubah dari "present" menjadi "late"
- Tetap dihitung sebagai hadir dalam attendance_rate

Field Duty Integration:
- Hanya field duty dengan status "approved" yang dihitung
- Jika ada field duty di tanggal tertentu, tidak perlu absen
- Field duty multi-hari akan mengisi semua tanggal dalam range
- Status di calendar: "field_duty"

Leave Integration:
- Hanya leave dengan status "approved" yang dihitung
- Jika ada leave di tanggal tertentu, tidak perlu absen
- Leave multi-hari akan mengisi semua tanggal dalam range
- Status di calendar: "leave"

Priority:
1. Field Duty (highest priority)
2. Leave
3. Attendance (present/late)
4. Absent (default jika tidak ada data)

Month Parameter:
- Format: YYYY-MM (contoh: 2026-04)
- Bisa query bulan lalu, bulan ini, atau bulan depan
- Bulan depan akan menampilkan data kosong (belum ada absensi)

Indonesian Locale:
- Day names: Senin, Selasa, Rabu, Kamis, Jumat, Sabtu, Minggu
- Month names: Januari, Februari, Maret, April, dst
- Menggunakan Carbon::locale('id')

=====================================
MOBILE APP INTEGRATION GUIDE
=====================================

Recommended Flow:
-----------------

1. Dashboard Screen:
   - Call GET /api/attendance/statistics (current month)
   - Display attendance_rate dan punctuality_rate
   - Show present_days, late_days, absent_days

2. Calendar View:
   - Call GET /api/attendance/recap?month=YYYY-MM
   - Use "calendar" object untuk render calendar
   - Color coding:
     * Green: present
     * Yellow: late
     * Red: absent
     * Blue: field_duty
     * Purple: leave

3. List View:
   - Call GET /api/attendance/recap?month=YYYY-MM
   - Use "details" array untuk render list
   - Show date, day_name, status, check_in, check_out

4. Detail View:
   - Call GET /api/attendance/detail/{date}
   - Show full detail dengan foto dan lokasi
   - Display working_hours

Month Navigation:
-----------------
- Previous month: month = current_month - 1
- Next month: month = current_month + 1
- Format: YYYY-MM

Caching Strategy:
-----------------
- Cache recap data per month
- Invalidate cache when:
  * User check in/out
  * Field duty approved
  * Leave approved
- Refresh on pull-to-refresh

Error Handling:
---------------
- 404: Data tidak ditemukan (show empty state)
- 401: Token expired (redirect to login)
- 422: Invalid month format (show error message)

Performance Tips:
-----------------
- Load current month first
- Lazy load other months on demand
- Use pagination for details array if needed
- Compress images before upload
